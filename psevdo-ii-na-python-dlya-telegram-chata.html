"Псевдо ИИ" на Python для Telegram чата

Demetrius Bellioz 19 июля 2020 Python

У нас с друзьями есть групповой чат в Telegram. Дабы разнообразить наше общение, я решил написать бота который случайным образом пишет сообщения в чат. В основе своих сообщений бот использует цепь Маркова и историю сообщений чата.

Пример небольшого диалога с участием бота:

1. Версия Python и используемые библиотеки

Python 3.8.3

Сторонние библиотеки:

pyTelegramBotAPI - Библиотека для работы с Telegram API.
Markovify - Отвечает за алгоритм цепи Маркова.
PySocks - Библиотека для поддержки Socks5 прокси.

Установка сторонних библиотек одной командой: pip install pytelegrambotapi markovify pysocks

Стандартные библиотеки:

Re - Для работы с регулярными выражениями.
Json - Для работы с JSON.
Random - Для генерации случайных чисел.

2. Создание бота в @Botfather и настройка

Как создать бота расписывать не буду, так как инструкций на эту тему в интернете полно. Например: https://tlgrm.ru/docs/bots#botfather

Настройка:

В config.json нужно прописать токен бота, ID чата и username вашего бота без @. ID чата можно узнать при помощи @myidbot, просто добавьте его в ваш чат и наберите команду /getgroupid.

3. Структура бота и команды

Бот состоит из четырех файлов:
bot.py - Код бота.
config.json - Настройки бота.
phrases.txt - История сообщений чата.
stopwords.txt - База слов-исключений.

Бот понимает всего две команды:
/stat - Узнать кол-во уникальных слов в базе. Чем больше уникальных слов в базе, тем лучше и разнообразней будут генерироваться сообщения.
/set_level - Установить уровень говорливости от 0 до 100. Например /set_level 30. Чем больше число, тем чаще бот будет писать в чат, чтобы отключить бота напишите /set_level 0.

4. Краткое описание алгоритма

После того как вы настроили бота и добавили его в чат, первое время он будет молча сидеть и логировать сообщения в файл phrases.txt. Как только база сообщений пополнится достаточным количеством уникальных слов, бот напишет свое первое сгенерированное сообщение.

Бот обрабатывает сообщения в зависимости от настроек /set_level. Например если /set_level равен 50, то бот будет обрабатывать каждое новое сообщение с вероятностью в 50% , если /set_level равен 100, то будет обрабатывать абсолютно каждое новое сообщение.

При обработке нового сообщения бот разбивает его на отдельные слова, и формирует список из этих слов, кроме слов которые находятся в базе слов-исключений stopwords.txt. Затем из этого списка слов случайным образом выбирает одно слово, и составляет список предложений из истории чата phrases.txt которые содержат это слово. И уже на основе составленного списка предложений генерирует свое сообщение.
